Übungsaufgaben JavaScript 1

Aufgabe 1

Basics

a) Truthy and Falsy (siehe z.B. developer.mozilla.org)

Schreiben Sie ein kurzes Programm, das zunächst eine Variable let x definiert ohne diese zu initialisieren. Verwenden Sie innerhalb einer if-Abfrage nur diese Variable (if(x)), um den Satz „x hat einen der folgenden Werte: false, 0, -0, "", null, undefined oder NaN.“ nur dann in der Konsole auszugeben, wenn die Aussage auch zutreffend ist.

b) JavaScript ist eine dynamisch typisierte Sprache

Geben Sie mit JavaScript eine Initialisierung für die Variablen x und y an, so dass danach gilt (x != y) && (x + 1 == y + 1)

c)[Für Fortgeschrittene] Häufig trifft man auf die sogenannte Pfeilfunktion-Notation in JavaScript. Aber wofür sollte man diese lieber nicht verwenden?

Recherchieren Sie z.B. auf https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Functions/Arrow_functions?retiredLocale=de

Beispiel Pfeilfunktion-Notation (=>):

let arrowFunction =(x)=>{
      return x+1;
    }

Aufgabe 2

Eventhandler für Benutzerereignisse

a) Informieren Sie sich in den bekannten Quellen über HTML-Ereignisattribute, die für HTML-Elemente angegeben werden können (z. B. onclick, onmouseover, onmouseout, onload, ...):

  • W3Schools HTML DOM Events
  • SelftHTML Eventhandler

Erstellen Sie ein HTML-Dokument mit verschiedenen Elementen. Verknüpfen sie die Elemente mit Ereignissen (je Element ein Ereignis) und schreiben Sie entsprechende Javascript-Funktionen (sog. Eventhandler), die bei Eintritt des Ereignisses aufgerufen werden. Ändern Sie in den JavaScript-Funktionen jeweils den Inhalt des Elements und dessen Hintergrundfarbe.

Hinweis: Sie sollten den HTML-Elementen IDs geben, über diese IDs die Elemente im !JavaScript ansprechen (document.getElementById()) und dann deren Attribute ändern (innerHTML, style). Fragen Sie auch die Suchmaschine Ihrer Wahl.

Schreiben Sie Eventhandler für mindestens acht verschiedene Ereignisse.

b) Probieren Sie aus, was passiert, wenn mehrere verschiedene, konkurrierende Eventhandler an ein und dasselbe HTML-Element geknüpft werden (z. B. onclick, onmouseover).

Aufgabe 3

Validierung von Formularen

Informieren Sie sich über HTML-Formulare und die verschiedenen Typen des input-Elements: SelfHTML Input. Erstellen Sie ein HTML-Formular, das persönliche Daten entgegennimmt:

  • Name, Vorname
  • Geburtsdatum
  • Adresse
  • Mailadresse
  • Telefonnummer

Verwenden Sie jeweils die spezialisierten Typen des input-Elements. Formatieren Sie das Formular mit Bootstrap so, dass es gefällig aussieht, also in einem geschäftlichen Kontext verwendet werden könnte.

Erstellen Sie eine (oder mehrere) JavaScript-Funktionen, die beim Absenden des Formulars (Klick auf einen submit-Button), folgende Daten validieren:

  • Entsprechend des Geburtsdatums muss das Alter der Person bei mindestens 18 Jahren liegen.
  • Die Mailadresse muss der Form xxxx@xxxx.xxxx folgen, wobei xxxx beliebiger, exakt vierstelliger, Text sein darf.

Informieren Sie sich über die dafür geeigneten Funktionen der JavaScript-API ebenfalls online. Den Inhalt eines input-Elements kann man oft über dessen Eigenschaft value auslesen.

Falls Daten nicht den Vorgaben entsprechen, sollen die jeweils betroffenen input-Elemente einen roten Rahmen erhalten und beim Überfahren mit der Maus soll ein Hinweistext angezeigt werden, wie das erwartete Format der Eingabe lautet. Falls die Daten korrekt eingegeben wurden, soll in der HTML-Seite unter dem Formular eine Erfolgsmeldung eingeblendet werden.

Hinweis: Überlegen Sie, wie das sehr einfach mit einer CSS-Eigenschaft gesteuert werden kann.

Zusatzaufgabe Adressvalidierung mit Onlinedienst:

Finden Sie heraus, wie mit einem Onlinedienst überprüft werden kann, ob die eingegebene Adresse tatsächlich existiert und nehmen Sie diese Prüfung in Ihre Datenvalidierung mit auf.

Aufgabe 4

Eigenschaften eines Events auswerten

Bislang haben Sie nur Elementwerte ausgelesen und geändert. In vielen Kontexten ist es jedoch erforderlich, Informationen über das eingetretene Ereignis selbst zu erlangen (z. B. Mausposition, gedrückte Taste, ...). Dies erfolgt, indem die Eventhandlerfunktion um einen Parameter event erweitert wird, der von der Laufzeitumgebung beim Eintritt des Ereignisses gesetzt wird. Auf die Eigenschaften dieses Parameters kann dann im Eventhandler zugegriffen werden.

Informieren Sie sich z. B. hier:

  • W3Schools MouseEvent pageX
  • W3Schools TryIt

Erstellen Sie eine HTML-Datei mit zwei ineinander geschachtelten div-Elementen der Größen 600 x 400 Pixel und 30 x 30 Pixel.

a) Sobald in das äußere Element geklickt wird, soll sich das innere Element (dessen Mittelpunkt) an die Position des Mausklicks bewegen (nicht springen). Falls der Mausklick so nah am Rand des äußeren Elements stattfindet, dass das innere beim Bewegen über den Rand gelangen würde, soll es nur bis zum Rand bewegt werden (es bleibt immer vollständig innerhalb des Spielfeldes). Recherchieren Sie die notwendigen Funktionen der JavaScript-API selbständig.

Verwenden Sie für die Bewegung CSS-Transitionen. Sie müssen also den Mausklick abfangen und dann die neue Position berechnen und die entsprechenden style-Attribute des Elements per !JavaScript anpassen.

b) Ändern Sie Ihre Lösung zu a) (oder eine Kopie davon) so ab, dass Sie das innere Element frei mit den Cursortasten steuern können. Es soll sich ebenfalls nicht über den Rand des Spielfeldes bewegen können. Informieren Sie sich selbständig über die passenden Events und deren Verarbeitung.
